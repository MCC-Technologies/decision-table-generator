# Contributing

Thank you for your interest in contributing to Decision Table Generator. Contributions of all kinds are welcome — bug reports, feature suggestions, documentation improvements, and code.

## Getting Started

1. Fork the repository
2. Create a branch for your change: `git checkout -b feat/your-feature-name`
3. Make your changes
4. Run the app and verify everything works: `flutter run`
5. Submit a pull request against the `main` branch

## Reporting Bugs

Open an issue and include:
- A clear description of the problem
- Steps to reproduce it
- Expected vs actual behavior
- Flutter and Dart SDK versions (`flutter --version`)
- Platform (Android, iOS, Linux, macOS, Windows, Web)

## Suggesting Features

Open an issue with the `enhancement` label. Describe the use case, not just the feature — it helps with prioritization.

## Pull Requests

- Keep pull requests focused on a single change
- Match the existing code style
- If your change affects user-facing behavior, update the README accordingly
- Link the related issue in your pull request description if one exists

## Code Style

This project follows standard Flutter/Dart conventions. Run the following before submitting:

```bash
flutter analyze
dart format .
```
